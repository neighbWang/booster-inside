WebP 图片压缩

Cwebp

与 pngquant 不同，cwebp 采用的是 BSD License，与 Apache License 并不冲突，所以 booster-task-compression-cwebp 内置了从 Google 官方提供的下载地址：https://storage.googleapis.com/downloads.webmproject.org/releases/webp/index.html 下载的 cwebp 二进制可执行文件 64 位版本。

实现思路

与 PNG 图片压缩 的思路一样，也是在 mergeRes 和 processRes 任务之间插入 WebP 压缩任务，如下图所示：

由于 WebP 格式在 14 <= minSdkVersion <= 17 不支持带 alpha 通道的图像，所以，针对 AAPT2, alpha 两个维度，将 task 分成了4 种：

需要注意的是：如果应用是发布在 Google Play 上，应用的启动图标必须是 PNG 格式，否则 Google Play 不会接受。

如何使用

开启 WebP 压缩只需要引入 booster-task-compression-cwebp 即可，如下所示：

build.gradle

buildscript {
    ext {
        kotlin_version = '1.3.31'
        booster_version = '1.6.0'
    }
    repositories {
        mavenLocal()
        mavenCentral()
        google()
        jcenter()
        maven { url 'https://oss.sonatype.org/content/repositories/public/' }
        maven { url 'https://oss.sonatype.org/content/repositories/snapshots/' }
    }
    dependencies {
        classpath 'com.android.tools.build:gradle:3.5.0'
        classpath "org.jetbrains.kotlin:kotlin-gradle-plugin:$kotlin_version"
        classpath "com.didiglobal.booster:booster-gradle-plugin:$booster_version"

        /* 👇👇👇👇 引用这个模块 👇👇👇👇 */
        classpath "com.didiglobal.booster:booster-task-compression-cwebp:$booster_version"
    }
}

与 PNG 压缩混用

由于只有 minSdkVersion 在 18 以上才能完全使用 WebP 压缩，所以，为了解决 minSdkVerion 在 18 以下，部分图片无法使用 WebP 压缩的情况，这时候就需要用 booster-task-compression-pngquant 来弥补，当 booster-task-compression-pngquant 与 booster-task-compression-cwebp 共存时，task 的依赖关系是这样的：

同时启用 PNG 和 WebP 压缩只需要引入 booster-task-compression-cwebp，booster-task-compression-pngquant 和 booster-pngquant-provider 即可：

build.gradle

buildscript {
    ext {
        kotlin_version = '1.3.31'
        booster_version = '1.6.0'
    }
    repositories {
        mavenLocal()
        mavenCentral()
        google()
        jcenter()
        maven { url 'https://oss.sonatype.org/content/repositories/public/' }
        maven { url 'https://oss.sonatype.org/content/repositories/snapshots/' }
    }
    dependencies {
        classpath 'com.android.tools.build:gradle:3.5.0'
        classpath "org.jetbrains.kotlin:kotlin-gradle-plugin:$kotlin_version"
        classpath "com.didiglobal.booster:booster-gradle-plugin:$booster_version"

        /* 👇👇👇👇 引用这三个模块 👇👇👇👇 */
        classpath "com.didiglobal.booster:booster-task-compression-cwebp:$booster_version"
        classpath "com.didiglobal.booster:booster-task-compression-pngquant:$booster_version"
        classpath "io.johnsonlee.booster:booster-pngqaunt-provider:1.1.0"
    }
}

自定义压缩质量

booster-task-compression-cwebp 支持自定义的参数如下表所示：

选项 描述
booster.task.compression.cwebp.option.quality 压缩质量 (默认 80)

通过 gradle.properties 配置

gradle.properties

booster.task.compression.cwebp.option.quality=75

通过命令行配置

$ ./gradlew assembleDebug -Pbooster.task.compression.cwebp.option.quality=75
